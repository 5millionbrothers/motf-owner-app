# 사장님·운영팀 UI 운영 기준

## 현재 기준

`public/owner/index.html`과 `public/owner/owner-data.js`가 현재 배포 화면의 기준 원본이다.
`next.config.ts`가 루트 주소를 이 화면으로 연결한다. Supabase 스키마와 함수의 원본은
`motf-database` 저장소에서만 관리한다.

## 수정 원칙

1. 현재 배포 기능 수정은 `public/owner/`에서 한다.
2. 동일한 기능을 `src/`에 동시에 복제하지 않는다.
3. 화면에 예시 회원, 거래, 매출을 운영 데이터처럼 표시하지 않는다.
4. 회원, 업장, 거래, 채팅, 문의는 Supabase 조회 결과만 표시한다.
5. DB 변경은 `motf-database`에 번호가 붙은 SQL로 먼저 기록한다.

## Next.js 이전 완료 조건

- 사장님 인증과 입점 상태 확인
- 업장, 객실·상품, 이미지 관리
- 숙소 예약과 공판장 주문 처리
- 사장님 채팅과 운영팀 읽기 전용 모니터링
- 문의 처리와 관리자 회원·파트너 관리
- 역할별 권한 및 모바일 화면 회귀 테스트

모든 조건을 충족하면 Vercel 미리보기에서 검증한 뒤 `/owner/index.html` 리다이렉트를 제거하고
`src/`를 새 기준 원본으로 전환한다.
